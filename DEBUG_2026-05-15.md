## Результаты исправлений

### ✅ 1. Автозапуск с Windows
- Ярлык `Spee4ka.lnk` в `Startup\` **СЕЙЧАС есть**, target верный (`$LOCALAPPDATA\Spee4ka\Spee4ka.exe`)
- Возможные причины пропадания между перезагрузками:
  - Windows Update / антивирус мог удалить
  - Случайный запуск деинсталлятора (он стирает ярлык, см. `installer_build\spee4ka_setup.iss` строка 89-96)
  - Чистильщики типа CCleaner
- **Действие пользователя:** если опять пропадёт — заново зайти в Настройки и нажать Сохранить. Долгосрочно — нужно дублировать через реестр `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` (стабильнее ярлыка), но пока не делал.
- Также: проверь в Windows → Параметры → Приложения → Автозагрузка — там Спичка должна быть в списке и переключатель = Вкл.

### ✅ 2. Окно "Ввести ключ активации"
- Корневая причина: `tkinter.TclError: Can't find a usable init.tcl` — встроенный Python не находил Tcl runtime, потому что `TCL_LIBRARY` не выставлен в subprocess
- **Fix:** `activation_window.py` теперь устанавливает `TCL_LIBRARY` и `TK_LIBRARY` ДО `import tkinter`, + `os.add_dll_directory` для `DLLs/`
- **Дополнительно:** `main.py` теперь делает явный `sys.path.insert(0, ROOT)` чтобы `license_manager` и `crypto_utils` всегда находились (Spee4ka.exe-лаунчер не гарантировал это)

### ✅ 3. "Лицензия не активна" при работающей программе
- Корневая причина: когда `from license_manager import check` падает с `ImportError`, функция `_check_license` возвращала `True` (правильно — пропускаем проверку) **но не обновляла глобал `_license_valid`** → меню показывало "✗ не активна"
- **Fix:** в `except ImportError` теперь `_license_valid = True` — UI становится консистентным

### 📖 4. Яндекс API расход — объяснение (не баг)

**Почему Яндекс используется хотя ты на "оффлайн-виспере":**
- Режим `offline_first` означает **"Whisper сначала, Яндекс если Whisper не справился"** — это НЕ "только Whisper"
- Когда Whisper возвращает пустоту (короткие/тихие записи) — код делает fallback на Яндекс STT (логи `Yandex replay`)
- **YandexGPT-полировка** — отдельный API. При `polish=True` КАЖДЫЙ результат Whisper отправляется в YandexGPT для расстановки знаков препинания и убирания слов-паразитов. Это и есть основная статья расхода Яндекс API в offline-режиме.

**Sync vs Async (изменение биллинга):**
- В сессии 8 (11 мая) поменяли `audio_processing_type` с `REAL_TIME` (= синхронное в биллинге) на `FULL_DATA` (= асинхронное в биллинге) — для лучшего качества распознавания
- FULL_DATA дешевле: 5376 сек × ~0.01₽/сек = 54.30₽ vs sync 1081 × 15-сек блок × ~0.16₽ = 175.77₽
- То есть переход с sync на async сэкономил тебе деньги, не потратил больше

**Чтобы НЕ платить Яндексу вообще:**
1. В Настройках убрать галку "Полировать GPT" — это уберёт самую большую статью расхода
2. Удалить Яндекс API ключи (тогда fallback на Yandex не сработает)
3. Останется чистый Whisper-режим, бесплатно

### ✅ 5. Ctrl+V в Настройках с русской раскладки
- Раньше в `settings_window.py` не было обработки Ctrl+V вообще — Tk полагался на дефолтный bind по keysym 'v', но на русской раскладке keysym = 'м'
- **Fix:** в `_styled_entry` теперь биндится `<Control-v>`, `<Control-V>` И `<Control-KeyPress>` с проверкой `event.keycode == 86` (физический код клавиши V на Windows независимо от раскладки)

### ✅ 6. Зависание записи
- Если `keyboard`-хук пропускает событие отпускания клавиши, запись идёт бесконечно
- **Fix:** в `on_press` запускается watchdog-таймер на 60 секунд. Если запись длится дольше — `_watchdog_force_release()` принудительно вызывает `_do_release()`. Таймер отменяется при нормальном отпускании клавиши.
- В логе при срабатывании watchdog увидишь: `Watchdog: recording exceeded 60.0s — force-stopping`
